## `$ ls ./projects/ --featured`
 
<details>
<summary><b>🔬 AI Code Reviewer — Multi-Agent LLM Code Analysis Pipeline</b></summary>
<br/>
> **Problem:** Code review is slow, inconsistent, and doesn't scale. Manual review misses security vulnerabilities and architecture anti-patterns.
 
**Architecture:** Parallel fan-out multi-agent system using **LangGraph** — four specialized agents run concurrently: Static Analysis, Security, Style, and Architecture reviewers, with a final synthesis node.
 
**Stack:** `Python` · `FastAPI` · `LangGraph` · `async SQLAlchemy` · `Alembic` · `AWS RDS (PostgreSQL)` · `Docker` · `AWS EKS` · `Terraform` · `GitHub Actions` · `Prometheus`
 
**Key Features:**
- Parallel agent execution with LangGraph fan-out pattern
- Async FastAPI backend with SQLAlchemy + Alembic migrations
- Full MLOps: Dockerized, EKS-deployed, CI/CD via GitHub Actions
- Prometheus instrumentation for observability
[![GitHub](https://img.shields.io/badge/View_Repo-161B22?style=flat-square&logo=github&logoColor=white)](https://github.com/nishan129/Github_code_reviewer)
![Status](https://img.shields.io/badge/Status-Production-00D9FF?style=flat-square)
 
</details>
<details>
<summary><b>🎙️ Real-Time Voice AI Backend — Hindi/Hinglish Voice Agent Platform</b></summary>
<br/>
> **Problem:** Conversational AI for Indian languages requires low-latency STT/TTS that works reliably in production without crash loops.
 
**Architecture:** FastAPI + WebSocket server with a `ModelRegistry` singleton loaded once at startup via `lifespan` hook. LangGraph orchestrates dialogue flow with session-aware state.
 
**Stack:** `Python` · `FastAPI` · `WebSocket` · `Wav2Vec2 (STT)` · `XTTS-v2 / Supertonic (TTS)` · `LangGraph` · `Sarvam AI` · `Redis` · `Docker`
 
**Key Features:**
- Singleton model registry (no per-connection model load — eliminates crash-restart loops)
- Async streaming session management with Redis
- Hindi/Hinglish STT with Groq Whisper fallback
- Production-ready lifespan management
![Status](https://img.shields.io/badge/Status-Active-00D9FF?style=flat-square)
 
</details>
<details>
<summary><b>📚 RAG Pipeline on GCP — Cloud-Native Retrieval-Augmented Generation</b></summary>
<br/>
> **Problem:** Build a scalable, cloud-native RAG system with proper vector search, CI/CD, and infrastructure-as-code.
 
**Architecture:** Cloud Run microservice backed by Qdrant vector DB. Terraform manages all GCP infrastructure. Cloud Build handles CI/CD. Sentence Transformers (`all-MiniLM-L6-v2`, 384-dim) for embeddings.
 
**Stack:** `Python` · `GCP Cloud Run` · `Qdrant` · `Sentence Transformers` · `Terraform` · `Cloud Build` · `Artifact Registry` · `VPC Connector`
 
**Key Features:**
- Full IaC with Terraform (GCP project: `us-central1`)
- Embedding pipeline with dimension-aware vector schema
- VPC-secured Cloud Run deployment
- Automated CI/CD via Cloud Build
![Status](https://img.shields.io/badge/Status-Deployed-00D9FF?style=flat-square)
 
</details>
<details>
<summary><b>🛡️ Brand Guardian — LLM-Powered Compliance QA Pipeline</b></summary>
<br/>
> **Problem:** Manual brand compliance review is expensive and doesn't scale across audio/video assets.
 
**Architecture:** LangGraph orchestration pipeline. Audio transcription → semantic embedding → vector similarity search against brand guidelines → LLM compliance verdict.
 
**Stack:** `Python` · `LangGraph` · `Groq Whisper` · `Groq LLaMA 3.3 70B` · `HuggingFace all-MiniLM-L6-v2` · `Qdrant Cloud`
